UTS Website Migration


An iterative approach to migrate a large university website by keeping some existing website content pages but served from the new website CMS.

After exploring few options, oEmbed stood right fit given the minimal work required and also the benefit oEmbed brings to avoid CORS.

I have created a Drupal module to oEmbed Drupal pages. It uses Drupalsubrequest and the proceesing the response with various rules and returning the valid oEmbed response. Please feel free to view the Drupal code source code for oEmbed in my GitHub repository.

I have spiked using Declarative Shadow and Dynamic Declarative Shadow with oEmbed so there is no conflict with Drupal CSS and JS with Optimizely. With Dynamic Declarative shadow it process in the server side and makes implementation clean and easier.

Dynamic Declarative Shadow also overcomes the issues that might arise from hidden HTML of embed pages, there by causing SEO impact. There are some spike works in my GitHub reppository to demonstrate Dynamic Shadow Dom and Shadow Dom. Please feel free to check.

Apart from this i have been helping Team with Test Automation, Analysis redirects for SEO, exploring and spike future content migration from Drupal entities especially from Content Type Node. Feel free to look at Drupal module to explore contents using API, Forms and also using a Content Explorer interface that allows to filter fields and list and download to a CSV format. You could build something similar using Drupal Views but it would require creating individual views manuallly for each content type and alsa adding extra view formatters if it is missing in your site.

Finally, given we only need the old Drupal site for oEmbed response and for internal purpose, i have come up with a Drupal module to restrict routes by IP with backend forms with configurable Routes and IP. You could achieve something like this on Varnish aswell but you might reach out to platform support for configuration help.

Another piece of work i have done is to help migrating Drupal assets. Given its a massive websites with thousands of images and documents, i have created a CLI using RUST to make the process of migrating assets easier and upto date. There is also a Drupal module to get the latest assets files data from Drupal. Feel free to look at the Drupal module for Access Restriction by Routes, CLI and other Drupal modules in my GitHub repository.
